1. 项目背景与任务概览

本部分阐述了传统二维地图在表达三维地理空间时的局限性，以及重度 3D GIS 系统在 Web 端面临的性能与跨域痛点。本项目旨在提供一个轻量、高效的解决方案。

⚠️

传统痛点 (Problem)

  • ■ 二维地图无法直观展现高低落差与微观细节。
  • ■ 传统 3D GIS 依赖数百兆 GeoTIFF 或量化瓦片。
  • ■ Web 端高开销易致 CORS 跨域崩溃或内存过载。
🎯

项目任务 (Mission)

构建轻量级、高可用、数据同源的 Web 三维地形智能探索平台。

  • ✓ 前端局部 3D 地形秒级重建
  • ✓ AI 语义检索与 2D/3D 联动
  • ✓ GPU 级抗锯齿等高线与水文仿真

3. 核心算法与模型解析

本节展示了系统实现轻量化渲染的三个核心数学与图形学算法。通过前端内存解算，摆脱了对云端高密数据的重度依赖。

空间高程网格上采样算法

为降低网络开销，云端仅返回 10x10 控制点。系统采用双线性插值在内存中将其映射为 128x128 高密网格。对于坐标点 (u, v)，在四角已知高度间执行三次线性插值：

f(R₁) ≈ [ (x₂ - x)/(x₂ - x₁) ] * f(Q₁₁) + [ (x - x₁)/(x₂ - x₁) ] * f(Q₂₁)

f(R₂) ≈ [ (x₂ - x)/(x₂ - x₁) ] * f(Q₁₂) + [ (x - x₁)/(x₂ - x₁) ] * f(Q₂₂)

f(P) ≈ [ (y₂ - y)/(y₂ - y₁) ] * f(R₁) + [ (y - y₁)/(y₂ - y₁) ] * f(R₂)

应用目标：实现低控制点在 CPU 内存中的高密度、零延迟网格拓扑转换。

自适应多尺度分形地貌生成

插值格网过于平滑，系统叠加了不同振幅和频率的连续插值噪声（FBM），模拟地质风化与水流侵蚀的岩石微观粗糙感：

f(P) = Σ [ aⁱ * n(2ⁱ * P) ] (i=0 to k-1)

// 高度自适应衰减指数设计
Detail = fbm(U, V) * 35.0 * min( 1.0, Height / 1000.0 )

应用目标：只在海拔高、坡度陡的尖峰处体现险峻细节，而在平原、浅滩处自动归零，保持地形平整。

GPU 像素级偏导数等高线

抛弃传统贴图方案，在 Fragment Shader 中利用 WebGL 硬件级空间微分函数 fwidth() 实时解算等高线，消除锯齿与变形拉伸：

df = fwidth(y / spacing)

// 利用 smoothstep 绘制 1.3 物理像素的一致线宽
lineAlpha = smoothstep(df * W, df * (W * 0.15), distance)

应用目标：彻底解决大视角或陡峭斜坡上因拉伸而产生的“斑驳、锯齿、模糊”技术硬伤。

5. 总结与反思

本项目深度整合了空间地理学、Web 图形学与大型语言模型技术，提供了一套具有强工程落地价值的三维地形引擎。

🏆 核心成果

  • 高可用轻量引擎： 利用 100 点控制网格并发与插值，彻底替代百兆级 DEM 瓦片，解决跨域与内存崩溃。
  • 实体封闭重构： 首创“边界顶点垂直投影算法”，配合沉积层 GLSL 着色器，完美修复 3D 侧面镂空穿模。
  • 深度 GPU 定制： 实现偏导数抗锯齿等高线、无遮挡 Z-Offset 标注控制及自适应水文学淹没颜色仿真。

🚀 改进空间

  • AI 代理网络穿透： 当前直连 DeepSeek API 易受本地网络策略影响，未来拟引入 Node.js 中间件托管凭证调度。
  • 法线扭曲修正： 极限垂直夸张 (exaggeration > 2.5) 下，陡峭顶点法线反射异变。需在 Shader 补充基于斜率的阻尼函数平滑高光。
  • 交互扩展： 增加剖面线切片绘制工具与多边形体积（土方量）推算功能。
